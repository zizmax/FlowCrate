  3. Open the app's Settings, then copy the Client ID and Client Secret into the fields below. Click Save Settings, then Test Spotify.

Flow State

Public Flow State posts work by default. For paid posts, Flow Crate automatically reuses your logged-in browser session when a post is paywalled. (Your computer may ask once for access to read those cookies.) If automatic detection can't work — most often on a headless box like a Raspberry Pi with no browser — use the Headless setup command below to sync your session from another device. Pasting a cookie by hand is a last resort.

Paste the cookie by hand (if you can't use the command below)

Flow State is a Substack custom domain, so the cookie that unlocks paid posts is connect.sid on www.flowstate.fm — not substack.sid.

  1. In the browser where you're logged in to your Flow State subscription, open flowstate.fm.
  2. Open developer tools (Cmd+Option+I in Chrome/Firefox; in Safari enable the Develop menu first) and go to the storage/cookies view (Application → Cookies in Chrome).
  3. Select the www.flowstate.fm site, find the cookie named connect.sid, and copy its value into the field above, then Save Settings.
Headless setup (No browser on the Flow Crate host)

When Flow Crate runs somewhere without your logged-in browser (like a Pi), it can't read your session automatically. Run this one-liner on the Mac/PC where you are logged in to Flow State — it reads your session cookies locally and pushes them here. Requires uv (install with curl -LsSf https://astral.sh/uv/install.sh | sh).

{% if cfg.api_token %} 
uv run --with browser-cookie3 --with requests python -c "import browser_cookie3 as b,requests as r;f=b.firefox;c={x.name:x.value for x in list(f(domain_name='flowstate.fm'))+list(f(domain_name='substack.com'))};resp=r.post('http://{{ local_hostname }}:{{ local_port }}/api/session',headers={'X-FlowCrate-Token':'{{ cfg.api_token }}'},json={'connect_sid':c.get('connect.sid',''),'substack_sid':c.get('substack.sid','')});print('Session synced - the Flow Crate Settings page updates automatically (refresh if it does not).' if resp.ok else 'Sync failed: '+resp.text)"
Reads Firefox by default — swap b.firefox for b.chrome or b.safari if you're logged in elsewhere.

Sonos & Siri (optional)

Play the latest post directly on a Sonos speaker, including from an Apple Shortcut ("Hey Siri, play Flow Crate"). Set an API token to enable the /api/play-latest endpoint.

Then say: "Hey Siri, Play Flow Crate". The shortcut is signed on this Mac, so it already has this install's address and token built in — nothing to edit.

Set up the Shortcut manually

The phone must be on the same Wi-Fi network as this computer.

On your iPhone, open Shortcuts, create a new shortcut named exactly Play Flow Crate (the name is the Siri phrase), and add these four actions:

  1. Get Contents of URL
    • URL: http://{{ local_hostname }}:{{ local_port }}/api/play-latest
    • Method: POST
    • Header: X-FlowCrate-Token = your API token (above)
    • No request body is needed — the endpoint accepts an empty body and plays immediately.
  2. Get Dictionary Value — key speak from the previous action's output.
  3. Speak Text — the Dictionary Value (always read aloud).
  4. Show Result — the Dictionary Value (shown in the Siri card).

Then say "Hey Siri, Play Flow Crate." Music starts on the configured Sonos room and Siri reads the post title, date, and track count.
